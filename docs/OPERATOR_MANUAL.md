# The Drink Exchange - Operator Manual

This is the runbook for opening, running, and closing the bar. Aimed at the venue manager / owner-operator.

## Daily setup (16:00)

1. Power on all displays. Each kiosk should auto-boot Chrome and load:
   - Main bar TV -> `https://<your-domain>/display?profile=main`
   - Entrance display -> `/display?profile=tape`
   - Tabletop screens -> `/display?profile=featured`
2. Tap each screen once to enable audio (the crash stinger needs a single user gesture).
3. On a laptop or phone, open `/admin/shifts` and click "Open shift". This resets per-shift counters.
4. Open `/admin/menu` and confirm every drink for tonight is marked Active.
5. Optional: schedule the night's crashes at `/admin/crash` (e.g. 21:00 25%/3min).

## Trading

- Watch `/dashboard` from your phone or back office to monitor revenue, top movers, and margin alerts.
- Trigger a crash manually at any time from `/admin/crash` -> Trigger Crash Now.
- Social trigger: post a webhook URL such as `POST /api/crash/webhook/<token>` (see `.env`'s `CRASH_WEBHOOK_TOKEN`) to fire a 25%/2min crash from automation.
- POS works offline. If WiFi drops, staff see "OFFLINE - queued N" and orders auto-send when the network returns.
- Margin floor (default 30% of cost) is enforced server-side; you'll see dropped drinks in `/dashboard` under "Margin alerts".

## Mid-shift adjustments

- `/admin/market` to nudge volatility, decay, or noise. Changes apply on the next 2 second tick.
- `/admin/menu` to override the live price of a drink (logged in `/admin/audit`).
- `/admin/staff` to add or deactivate staff PINs.
- `/admin/audit` for a full ledger of every state-changing action.

## Closing (around 02:00)

1. POS clears all open tabs (charge them out or void with manager PIN).
2. On `/admin/shifts`, click "Close shift". The Z-report is generated, persisted on the shift record, and dispatched via the configured receipt provider (console logs in dev, Twilio/Postmark when configured).
3. The next morning, the daily summary email is generated by `/api/dashboard/daily-summary` (call this from your cron / Inngest scheduler at 03:00 ACDT in production).

## BAS / accounting

- `/admin/shifts` -> "Download BAS CSV" exports every order with GST broken out. Suitable for quarterly reporting.

## Disaster scenarios

| Symptom | Action |
|---|---|
| Display shows OFFLINE indicator | Check WiFi. Display will resume automatically when SSE reconnects. Last-known prices remain on screen. |
| POS shows OFFLINE for >30s | Orders are queued in localStorage. If you have to leave and come back, the same iPad will replay the queue when reconnected. Do not clear browser data. |
| Wrong price charged | `/admin/audit` -> find order. Use Manager Void on POS, or `/admin/shifts` -> manual refund flow. |
| Stuck crash that won't end | `/admin/crash` -> "End crash now". The screens revert immediately. |
| Tick engine appears frozen | Check `/api/state/snapshot`. The server-side scheduler restarts when the Next.js process restarts. If running on Railway, restart the service. |

## Environment variables

```
DATABASE_URL          Postgres connection (optional in dev; required in prod)
PAYMENT_PROVIDER      simulated | stripe | square
STRIPE_SECRET_KEY     Stripe Terminal secret (when PAYMENT_PROVIDER=stripe)
RECEIPT_PROVIDER      console | twilio | postmark
TWILIO_ACCOUNT_SID    Twilio SID (when RECEIPT_PROVIDER=twilio)
TWILIO_AUTH_TOKEN     Twilio auth token
POSTMARK_TOKEN        Postmark server token (when RECEIPT_PROVIDER=postmark)
CRASH_WEBHOOK_TOKEN   Secret for the social crash webhook
TRADING_OPEN          Default 16:00 (Australia/Adelaide)
TRADING_CLOSE         Default 02:00
GST_RATE              Default 0.10
```
